# 쿠팡 셀러 알리미 앱 (Android)

한 앱에서:
- **계정(1~10)을 눌러 언제든 쿠팡 윙 접속** — 계정마다 로그인이 따로 저장돼요 (한 번 로그인하면 다음부터 바로)
- **새 주문·문의·반품 알림을 종류별 다른 소리로** 받기 (소리는 앱 안에서 직접 고름)
- 알림을 누르면 그 계정 윙으로 바로 이동

알림 데이터는 이미 돌고 있는 **중계 서버 → ntfy 토픽**에서 받아와요. (서버는 그대로 두면 됩니다.)

---

## APK 만들기 (GitHub 자동 빌드 — 도구 설치 불필요)

컴퓨터에 개발 도구 없이 폰만으로 설치할 수 있게, GitHub가 대신 APK를 만들어줘요.

1. **github.com** 가입/로그인
2. **New repository** 로 새 저장소 생성 (이름 아무거나, Private 가능)
3. 이 폴더의 **모든 파일**을 저장소에 업로드
   (웹에서 `Add file → Upload files` 에 폴더째 끌어다 놓기)
4. 저장소 상단 **Actions** 탭 → **Build APK** → **Run workflow**
5. 몇 분 뒤, 저장소 오른쪽 **Releases** → **쿠팡 셀러 알리미 APK (최신)** →
   **app-debug.apk** 를 **폰 브라우저로** 눌러 다운로드
6. 다운로드한 APK 탭 → "출처를 알 수 없는 앱" 경고 시 **이 출처 허용** → 설치

---

## 앱 사용법

1. 앱 실행 → 맨 아래 **"서버 알림 주소(토픽)"** 에 서버에서 쓰던 토픽 입력 → **토픽 저장**
   (예: `coupang-alim-xxxxxxxx` — 서버 config.json 의 ntfyTopic 과 똑같이)
2. **＋ 계정 추가** 로 계정 이름 넣기 (예: `1번-우리상점`). 필요한 만큼 1~10개
3. 위쪽 **"알림 받기 시작"** → 알림 권한 허용, 배터리 최적화 제외 허용
4. **계정을 탭하면 그 계정 쿠팡 윙**이 열려요. 처음 한 번 로그인하면 다음부터 그 계정은 로그인 상태로 열려요
5. **알림 소리**: 🛒주문 / 💬문의 / ↩️반품 각각 버튼으로 다른 소리 지정

> 새 주문/문의/반품이 생기면 5분 안에 종류별 소리로 알림이 오고, 누르면 해당 계정 윙으로 이동해요.

---

## 참고

- 여러 계정 로그인을 동시에 유지하는 건 기술적으로 완벽하진 않아요. 세션이 만료되면
  그 계정만 다시 로그인하면 됩니다 (다시 저장돼요).
- 배터리 최적화 제외를 허용해야 화면을 꺼도 알림이 안 끊겨요.
- 서버(중계기)는 지금처럼 계속 돌아가야 알림이 와요. 앱은 그 알림을 받아 보여주는 역할이에요.
